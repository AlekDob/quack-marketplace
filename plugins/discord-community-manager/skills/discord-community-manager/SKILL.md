# Discord Community Manager

Expert guidance for managing and growing Discord-based developer communities.

## Capabilities

1. Server Structure & Channel Architecture
2. Community Growth Strategy
3. Engagement & Activity Management
4. Moderation & Community Health
5. Beta Testing Coordination
6. Analytics & Metrics
7. Community Events & Programming
8. Crisis Management

## Server Structure

Recommended channel categories:
- ANNOUNCEMENTS: announcements, changelog, roadmap
- COMMUNITY: general, showcase, off-topic
- SUPPORT: help, troubleshooting, feature-requests
- BETA TESTING: beta-announcements, beta-feedback, beta-bugs
- RESOURCES: documentation, tutorials, faq

## Role Hierarchy

- Admin (full permissions)
- Moderator (moderation, no server settings)
- Beta Tester (beta channel access)
- Early Adopter (recognition)
- Active Contributor (engaged members)
- Member (default)

## Engagement Guidelines

- Daily: Monitor help channels, respond within 4 hours
- Weekly: Post changelog updates
- Bi-weekly: Feature community members
- Monthly: Review roadmap with community

## Moderation

- Be welcoming, assume good intent
- Be transparent about decisions
- Warning system: verbal > written > timeout > ban
- Separate user conflicts from product frustration
